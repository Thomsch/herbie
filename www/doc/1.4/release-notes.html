Herbie 1.4 Release Notes

The Herbie developers are excited to announce Herbie 1.4! This release focuses on who knows?.

Herbie automatically improves the accuracy of floating point expressions. This avoids the bugs, errors, and surprises that so often occur when working with floating point. Since our PLDI'15 paper, we've been hard at work making Herbie more versatile and easier to use.

Major features of this release

Egg Support: Herbie now uses the Egg library to simplify expressions. Egg is roughly thirty times faster than Herbie's existing simplifier, making Herbie with Egg roughly twice as fast overall. Egg is packaged for Windows, macOS, and Linux, so it should be transparently usable on all platforms.

Improvement to core algorithm

  • Simplification now uses the Egg library; Herbie's existing e-graph library has been spun out as the regraph library, which can be used by other Racket projects.
  • The interval arithmetic library has gained support for movability flags, which generalize the floating-point exactness and overflow flags. This significantly speeds up certain cases of sampling, where too much time is spent sampling invalid points.
  • The interval arithmetic library has been spun out as the rival library, which other Racket projects can now use. Several minor bugs in the interval library have been fixed, and support has been extended to more functions. Testing has been improved to catch such bugs in the future.
  • The core program evaluator now operates in batch mode, speeding up cases where Herbie generates thousands of program candidates.
  • An exponential behavior in the rewrite algorithm was fixed by adding a cache, eliminating one common cause of timeouts.
  • Herbie's "final simplify" step no longer does constant folding, eliminating a rare source of accuracy regression.

Usability improvements

  • Misleading sentences were stricken from the documentation, which should now be a bit clearer and more approachable.
  • If a precondition is unsatisfiable, Herbie now names the correct variable when reporting the error.
  • Herbie now tries just a little hard to select simpler expressions when optimizing for accuracy. We've heard a lot of feedback about Herbie's confusing and weird results. This change does not totally fix the issue, but should help a bit.
  • Herbie now aggregates profile information across a whole report and presents it in an interactive, JavaScript-enabled form on the "Metrics" page. This should help the developers continue to speed up Herbie.
  • Some "internal" measures that Herbie presents on its report page now show the correct data.
  • The :spec property and let* construct are now supported.

Code Cleanup

  • Many obsolete functions and data structures have been removed. This makes Herbie easier to develop. A cost of this change is that pre-1.4 seeds will produce different results on Herbie 1.4 due to switching to a standard random-number API.
  • Herbie's support for multi- and mixed-precision operations continues to improve, though there is still more work to do before this task can be called "done".
  • Herbie's code base was reorganized. The increasingly-important web version is now spread across multiple files with stronger interfaces between them.
  • Herbie now uses Github Actions, not Travis, as its continuous integration service. Right now the effects are minor, but we hope to leverage Github Actions more in the future.
  • Herbie's support for complex number arithmetic has been extracted to an external library. The support remains in beta, but will no longer be a standard part of Herbie. In the future we hope to provide a standard API that complex number arithmetic can leverage, but right now the divorce is only partially complete.
  • Several herbie dependencies have been spun out as libraries, including the C compiler (fpbench), the e-graph implementation (regraph), and the interval library (rival). Separating out these components will make them easier for others to use.

Try it out!

We're excited to continue to improve Herbie and make it more useful to scientists, engineers, and programmers around the world. We've got a lot of features we're excited to work on in the coming months. Please report bugs, join the mailing list, or contribute.

If you find Herbie useful, let us know!
